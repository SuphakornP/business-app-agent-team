# Phase Gates

## Gate 0: Intake Complete

- Problem statement exists.
- Primary user is identified.
- Current workflow is described.
- Pain point is explicit.
- Initial success metric exists.

## Gate 1: Discovery Approved

- Critical clarifying questions are answered or escalated.
- Key assumptions are documented.
- Major risks are documented.
- Use cases are grounded in real scenarios.

## Gate 2: Scope Approved

- Must Have, Should Have, Later, and Out of Scope are explicit.
- Phase 1 is small enough to implement and test end to end.
- Critical open questions do not affect Phase 1 feasibility.

## Gate 3: Flow Approved

- User journeys are explicit.
- Screen flow is explicit.
- Empty, error, loading, and success states are covered.
- Permission-sensitive flows are called out.

## Gate 4: Technical Blueprint Approved

- Architecture boundary is clear.
- Data model maps back to flow and requirements.
- API contracts, permissions, and integrations are documented.
- Human approval items are resolved or escalated.

## Gate 5: Implementation Ready

- Tasks are split into small units.
- Acceptance criteria are complete.
- Test plan is ready.
- High-risk changes have approval.

## Gate 6: Delivery Ready

- Implementation evidence exists.
- Tests or checks pass.
- Reviewer findings are resolved or accepted.
- Docs are updated.
- No unresolved critical risk remains.
