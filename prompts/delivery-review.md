# Delivery Review

Review whether the project is ready to deliver.

Inputs:

- Approved requirements
- Implementation evidence
- Test results
- Review findings
- Risk register
- Open questions

Return:

- Delivery Summary
- Completed Scope
- Tests and Checks
- Known Limitations
- Risks
- Follow-ups
- Approval Recommendation

Do not mark delivery ready if critical risk, missing implementation evidence, or failed tests remain.
