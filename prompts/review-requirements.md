# Review Requirements

Review the requirement artifacts for ambiguity, missing user roles, missing validation, missing acceptance criteria, weak scope boundaries, and unapproved assumptions.

Inputs:

- `docs/product/04-scope.md`
- `docs/product/05-requirements.md`
- `docs/ux/01-user-journey.md`
- `docs/ux/02-screen-flow.md`

Output:

- `docs/qa/04-risk-review.md`
- `.agent-team/agent-results/qa-risk-reviewer.md`

Return an approval recommendation: `approve`, `approve with changes`, or `reject`.
